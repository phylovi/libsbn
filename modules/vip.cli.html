vip.cli¶

The vip command line interface has a nested subcommand interface. The documentation describes the vip command and then the various subcommands as nested sections.

vip¶

vip [OPTIONS] COMMAND [ARGS]...

benchmark¶

Do a benchmarking comparison to an MCMC run.

DATA_PATH is a path to a directory, which say is named X.

We assume that X contains:

X_out.t

an MCMC run on a fixed tree topology, and

X.fasta

a FASTA file with the same sequence data as used for MCMC.

vip benchmark [OPTIONS] DATA_PATH

Options

--branch-model <branch_model>¶

[default: split]

Options

split|psp

--scalar-model <scalar_model>¶

[default: lognormal]

Options

lognormal|tf_lognormal|tf_truncated_lognormal|tf_gamma

--optimizer <optimizer>¶

[default: simple]

Options

simple|bump

--step-count <step_count>¶

Number of gradient descent steps to take. [default: 5]

--particle-count <particle_count>¶

Number of particles to use for stochastic gradient estimation. [default: 10]

--thread-count <thread_count>¶

Number of threads to use. [default: 4]

--out-prefix <out_prefix>¶

Path prefix to which output should be saved.

Arguments

DATA_PATH¶

Required argument
